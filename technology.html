Technology

SciSpark is implemented in a Java and Scala environment. Although Spark also offers a Python environment, and we recognize that most scientist are comfortable with Python as a programming language, this environment was chosen to avoid the known latency issues related to the communication overhead involved with copying data from the worker JVMs to Python daemon processes in the PySpark environment. Furthermore we wish to keep as much computation as possible in memory but in the PySpark environment, the driver JVM writes results to local disk that are then read by the Python process. See our latest paper for more details.

Partition, Extract, Transform, Load

Scientific array-based data from the network Common Data Form (netCDF) and Hierarchical Data Format (HDF) files are loaded into SciSpark using a Partition, Extract, Transform and Load (PETaL) process from local files or from remote protocols like OPeNDAP. More specifically, the PETaL layer first partitions by time ( by space to be added) and distributes the extraction of the data, transformation of it into a data type usable in SciSpark and loading into of it into the SciSpark engine to the compute nodes.

sRDDs

In the same way Spark exploits Resilient Distributed Datasets (RDDs), SciSpark contributes and exploits the Scientific RDD (sRDD) that corresponds to a multi-dimensional array representing a scientific measurement (grid) subset by space, or by time. The RDD notion directly enables the reuse of array data across multi-stage operations and it ensures data can be replicated, distributed and easily reconstructed in different storage tiers, e.g., memory for fast interactivity, SSDs for near real time, and spinning disk for later operations.

SciSpark defines the Scientific Resilient Distributed Dataset (sRDD), a distributed-computing array structure that supports multidimensional data and processing of scientific algorithms in the MapReduce paradigm. This is currently achieved through a self-documented array class called sciTensor that defines the data in the multi-dimensional format that scientists are accustomed to.
SciSpark currently provides methods to create sRDDs that (1) loads data from network Common Data Form (netCDF) and Hierarchical Data Format (HDF) files into the Hadoop Distributed File System (HDFS); (2) preserve the logical representation of structured and dimensional data in ; and (3) create a partition function that divides the multidimensional array by time (to be expanded to space as well). sRDDs are cached (in memory) in the SciSpark engine support data reuse between multi-staged analytics.

SciSpark User Interface

One of the key components of SciSpark is interactive sRDD visualizations. To accomplish this SciSpark delivers a user interface through Apache Zeppelin. Zeppelin provides a notebook-type interface with an interpreter that allows for any language or data-processing-backend to be plugged in.
